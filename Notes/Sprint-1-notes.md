## Notes
- All work should be done on a separate branch, not on `main`.
- The `main` and `develop` branches are protected and cannot be pushed to directly.
- Every change must be merged using a pull request.
- At least one team member must review and approve a pull request.
- Team members should pull the latest `develop` branch before starting new work.
- Code should be tested locally before creating a pull request.
- Keep commits small and clear.
